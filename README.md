# NHANES accelerometry cardiometabolic -- SD4H_at_ICML_2026

Repository for the SD4H workshop @ICML 2026.

This repo accompanies the submission *NHANES Accelerometer Cardiometabolic Benchmark* — a comparison of tabular models (Ridge, XGBoost, MLP, and TabPFN) for predicting cardiometabolic biomarkers from NHANES physical-activity and covariate data, with conformal prediction intervals reported alongside point accuracy.

### A note on the two notebooks

`*_orig.ipynb` is the original analysis exactly as exported from Colab. It will not preview on
GitHub ("Invalid Notebook: the `state` key is missing from `metadata.widgets`") because Colab
writes a `metadata.widgets` block that GitHub's renderer rejects.

`*_clean.ipynb` is the **same notebook with the `metadata.widgets` block removed** so it renders
on GitHub. No cells, outputs, or results were changed — only the offending notebook-level metadata
was stripped. Use the clean version for browsing on GitHub; either runs identically in Colab.

## Data

The analytic dataset (`data/nhanes_analytic.csv`) is generated by `R/01_build_analytic_sample.R`,
which derives accelerometer summaries and covariates and joins them to the biomarker outcomes.

**Features**

- Activity: `TAC`, `TLAC`, `ST`, `MVPA`, `LIPA`, `WC`
- Numeric covariates: `RIDAGEYR`, `BMXBMI`, `INDFMPIR`, `BPXSY1`, `DR1TKCAL`, `DR1TCARB`, `DR1TTFAT`, `DR1TPROT`, `DR1TFIBE`
- Categorical: `RIAGENDR`, `RIDRETH1`, `SMQ020`

**Outcomes** (log-transformed for modeling)

- HbA1c (`LBXGH`)
- Triglycerides (`LBXTR`)
- CRP (`LBXCRP`)

## Running the analysis (Google Colab)

The notebook is written to run in Colab. Two manual steps are required before the model cells will run.

### 1. Mount Drive and upload the analytic dataset

The first cell mounts Google Drive and then **prompts you to upload `nhanes_analytic.csv`**.
When the file picker appears, select `data/nhanes_analytic.csv` from this repo. The notebook moves
it to `/content/data/nhanes_analytic.csv`, which the rest of the pipeline reads from.

### 2. Accept the TabPFN license and export a token

TabPFN requires a one-time license acceptance before its weights/inference can be used. In a Colab
(headless) session the interactive browser prompt is unavailable, so you must supply a token:

1. Go to <https://ux.priorlabs.ai>, sign in, and accept the license under the **License** tab.
2. Open **Account → Access Tokens**, click **Unmask**, and copy your personal token.
3. Export it in the notebook before the TabPFN cells run:

   ```python
   import os
   os.environ["TABPFN_TOKEN"] = "your_token_here"
   ```

The token is cached after first use, so this is a one-time step per environment. The token grants
full access to your Prior Labs account — keep it private and never commit it.

Once both steps are done, run the remaining cells top to bottom to reproduce the benchmark tables
(R² and conformal coverage per model and outcome, plus subgroup coverage).
